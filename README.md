# MPTCP Internet Combine
Combine multiple internet links into a single faster connection using MPTCP, WireGuard,
sing-box and socat. This repository provides simple installer scripts for a server (VPS)
and client to get started quickly.

**WARNING:** experimental scripts — use at your own risk and review scripts before
running them on production systems.

**Requirements**
- Linux VPS with a public IPv4 address (Debian/Ubuntu recommended)
- sudo/root access
- Kernel or userland support for MPTCP (the installer uses `mptcpize` in services)
- Open port for WireGuard (default: `51820/udp`) if behind cloud firewall

Quick start
1. Clone the repo and open the server installer:

```bash
git clone https://github.com/mgi24/MPTCP-Internet-Combine.git
cd MPTCP-Internet-Combine
nano serverinstall.sh   # edit variables at top (see notes below)
```

2. Run the server installer (on the VPS):

```bash
sudo bash serverinstall.sh
```

3. The installer prints a WireGuard client config at the end — copy that output.

4. On the client machine, paste the WireGuard client config into `clientinstall.sh` and
	 edit other client variables if needed, then run:

```bash
sudo bash clientinstall.sh
sudo bash clientup.sh
```

5. Start/stop helpers

```bash
sudo bash serverup.sh     # start server services
sudo bash serverdown.sh   # stop server services
sudo bash clientup.sh     # start client
sudo bash clientdown.sh   # stop client
sudo bash flushing.sh     # cleanup (server & client)
```

What the server installer does (high level)
- Enables basic MPTCP settings and IP forwarding
- Installs and configures WireGuard (`wg0`) and generates keys
- Sets MPTCP limits and adds MPTCP endpoint for signaling
- Installs `sing-box` and creates a local inbound (shadowsocks) used with `socat`
- Installs `socat` and creates a systemd service that runs `mptcpize run socat ...`
- Adds idempotent iptables NAT/forward rules and saves them persistently

Important variables (edit at top of `serverinstall.sh`)
- `PUBLIC_IP` — public IP address of the VPS (mandatory)
- `interface` — external network interface name (e.g. `eth0`) (mandatory)
- `Socat_Port` — public port for the mptcpized socat listener (default `8888`)
- `socat_internal_port` — local port where sing-box listens (default `8080`)
- `WG_PORT` — WireGuard listen port (default `51820`)

Notes about iptables and persistence
- The installer attempts to install `iptables-persistent` and `netfilter-persistent`
	non-interactively, then adds NAT (MASQUERADE) and FORWARD rules only if they
	don't already exist, and saves the current rules to `/etc/iptables/rules.v4`.
- If your distribution differs (or prompts during install), you may need to run
	`sudo netfilter-persistent save` or manually install `iptables-persistent`.

Client setup notes
- Paste the client WireGuard config generated by the server into `clientinstall.sh`.
- Run `sudo bash clientinstall.sh` to install dependencies and create the client
	WireGuard configuration, then `sudo bash clientup.sh` to bring the interface up.

Troubleshooting tips
- Check service logs:
	- `sudo journalctl -u sing-box -f` or `sudo systemctl status sing-box`
	- `sudo journalctl -u socat-bridge -f` (service name: `socat-bridge`)
- Verify WireGuard: `sudo wg show` and `ip a show wg0`
- Verify iptables rules: `sudo iptables -t nat -L -n -v` and `sudo iptables -L -n -v`
- Verify MPTCP endpoints: `ip mptcp endpoint show` and `ip mptcp limits show`

Security and final notes
- The installer generates WireGuard keys automatically on the server and prints the
	client private key in the generated client config — move that config to the client
	securely and remove any copies you don't need.
- Review and adapt firewall rules to your environment; using MASQUERADE alters
	outbound address translation.
